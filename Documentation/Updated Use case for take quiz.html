Title: take Quiz

Primary Actor: User:Student

Stakeholders and Interests:

User: The users in this case are the student taking the quiz

Precondition: The users are logged in the system as a student

Main Scenario:

  1. User logs into the system.
  2. If the credentials are verified the user is given access to the student dashboard
  3. Once inside the dashboard the student can select the quiz it would like to take
  4. The student navigates through the questions and selects the answer
  5. The student submits the attempt after completing the quiz
  6. The attempt can be viewed at a later date.

Extensions:

1a. If the user is not registered as a student, it is not given access to take a quiz.
